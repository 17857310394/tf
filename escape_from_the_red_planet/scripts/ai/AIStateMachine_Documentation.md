# AI 状态机系统文档

## 1. 系统概述

本系统使用 Godot 自带的 StateMachine 节点实现了一个完整的 AI 状态管理系统，包含巡逻、追击、攻击、逃跑、空闲等核心状态。系统采用模块化设计，清晰分离状态机逻辑与 AI 业务逻辑，便于扩展和维护。

## 2. 状态图

```

      ^                         |

```

## 3. 核心组件

### 3.1 AIStateMachine.gd

主状态机管理脚本，负责：

- 状态机初始化和状态转换
- AI 属性管理（巡逻点、移动速度、距离阈值等）
- 目标检测和伤害处理
- 状态转换逻辑

### 3.2 状态脚本

#### PatrolState.gd

- 巡逻逻辑：在预设巡逻点之间移动
- 到达巡逻点后短暂等待
- 自动切换到下一个巡逻点

#### ChaseState.gd

- 追击逻辑：向目标移动
- 追击速度比正常移动速度快 50%
- 持续计算与目标的距离

#### AttackState.gd

- 攻击逻辑：在攻击范围内停止移动
- 攻击冷却机制
- 对目标造成伤害

#### EscapeState.gd

- 逃跑逻辑：远离目标移动
- 逃跑速度比正常移动速度快 100%
- 定期改变逃跑方向

#### IdleState.gd

- 空闲逻辑：停止移动
- 随机决定是否开始巡逻

## 4. 使用指南

### 4.1 场景设置

1. 创建一个新场景，添加 `AIStateMachine.tscn` 实例
2. 添加一个玩家角色，并将其添加到 "player" 组
3. 确保场景中有 `NavigationRegion3D` 节点用于导航

### 4.2 配置参数

在 AIStateMachine 节点中可以调整以下参数：

| 参数               | 说明    | 默认值                                            |
| ---------------- | ----- | ---------------------------------------------- |
| patrol\_points   | 巡逻点数组 | \[-10,1,0], \[10,1,0], \[10,1,10], \[-10,1,10] |
| movement\_speed  | 移动速度  | 3.0                                            |
| chase\_distance  | 追击距离  | 10.0                                           |
| attack\_distance | 攻击距离  | 3.0                                            |
| escape\_distance | 逃跑距离  | 15.0                                           |
| health           | 当前血量  | 100                                            |
| max\_health      | 最大血量  | 100                                            |

### 4.3 控制方式

- **玩家移动**：使用方向键（上、下、左、右）控制玩家移动
- **AI 行为**：AI 会根据与玩家的距离和自身状态自动切换行为

## 5. 状态转换规则

| 当前状态   | 转换条件                      | 目标状态   |
| ------ | ------------------------- | ------ |
| Idle   | 发现目标且距离 < chase\_distance | Chase  |
| Idle   | 随机触发（1% 几率）               | Patrol |
| Patrol | 发现目标且距离 < chase\_distance | Chase  |
| Chase  | 距离 < attack\_distance     | Attack |
| Chase  | 距离 > escape\_distance     | Idle   |
| Attack | 距离 > attack\_distance     | Chase  |
| Attack | 血量 < 30%                  | Escape |
| Escape | 血量 > 70%                  | Idle   |
| Escape | 距离 < attack\_distance     | Attack |

## 6. 错误处理

系统包含以下错误处理机制：

1. **目标检测**：定期检查目标是否存在，不存在则重新查找
2. **状态恢复**：当状态转换失败时，会保持当前状态
3. **空值检查**：在所有关键操作前进行空值检查，避免空引用错误
4. **异常捕获**：捕获并处理可能的异常，确保系统稳定运行

## 7. 扩展指南

### 7.1 添加新状态

1. 在 `StateMachine` 节点下创建新的 `State` 节点
2. 为新状态创建对应的脚本文件
3. 实现 `_enter_state()`, `_exit_state()`, `_process(delta)` 方法
4. 在 `AIStateMachine.gd` 中添加状态转换规则

### 7.2 修改状态行为

可以通过修改各状态脚本的逻辑来调整 AI 行为：

- **PatrolState.gd**：调整巡逻点、等待时间
- **ChaseState.gd**：调整追击速度、行为逻辑
- **AttackState.gd**：调整攻击伤害、冷却时间
- **EscapeState.gd**：调整逃跑速度、方向改变频率
- **IdleState.gd**：调整空闲时间、随机行为概率

## 8. 测试建议

1. **基本行为测试**：观察 AI 在不同状态下的行为是否符合预期
2. **状态转换测试**：测试 AI 在不同条件下的状态转换是否正确
3. **边界情况测试**：测试 AI 在极限情况下的行为（如血量极低、距离边界等）
4. **性能测试**：测试多个 AI 实例同时运行时的性能表现

## 9. 性能优化

- **状态更新**：只在必要时更新状态，避免频繁状态转换
- **目标检测**：使用 `Area3D` 进行目标检测，减少距离计算
- **导航优化**：使用导航网格进行路径规划，避免逐帧计算路径
- **批处理**：对多个 AI 实例使用批处理更新
